claude-crew Roadmap

项目目标、版本规划、待办管理 — 始终围绕多项目管理和多人远程协作

使命

让团队在一个群聊里管理所有项目的 Claude Code。

不做万能桥接器，不做单人远程控制。做群聊协作工作台。

北极星指标

指标定义为什么重要
群活跃度一个群里每天有多少不同成员 @bot 发任务核心：多人在同一个群里协作
多项目并行度一个群里同时在跑的不同项目 bot 数量核心：多项目集群管理
手机操作比不碰终端就能完成的管理操作占比差异化：完全脱离电脑

v0.3 — 核心体验强化 NEXT

目标：解决群聊场景下的实际使用痛点，让日常使用更顺畅

Claude Code 命令透传

让用户在群里直接使用 Claude Code 的核心命令，无缝衔接终端体验

　
/new — 重置会话
下次调用不带 --continue，起一个全新的 Claude 会话。解决上下文膨胀变慢的问题
多项目管理体验优化
　
/compact — 压缩上下文
触发 Claude Code 的 context 压缩，不丢失关键信息的前提下释放空间
多项目管理性能
　
/model — 运行时切模型
在群里直接切换 bot 的模型（sonnet/opus/haiku），不用进配置菜单
效率
　
/effort — 调整思考深度
low/medium/high/max，按任务复杂度灵活切换
效率成本控制
　
/cost — 查看花费
显示当前 bot 的累计花费和 token 用量
成本控制
　
/memory — 查看项目记忆
显示项目的 CLAUDE.md 内容，了解 bot "记住了什么"
项目连续性
　
/status — bot 状态
单个 bot 的快捷状态：context 用量、模型、最近活动
可观测性

群聊体验优化

　
Telegram Thread 支持
每个 @bot 任务自动开一个话题 thread，进度和结果在 thread 里。主时间线保持干净，展开 thread 看细节。多项目并行时不再混乱
多项目管理多人协作高优先
　
任务队列
多人同时 @同一 bot 时排队执行，不再直接拒绝。群里显示队列位置："你是第 2 个，前面还有 1 个任务"
多人协作高优先
　
结果投票按钮
bot 结果自动带 ✅ 接受 / 🔄 重做 / 💬 讨论 按钮。团队成员投票，达到阈值自动执行下一步
多人协作高优先
　
Context 用量预警 + 自动压缩
context 超 80% 主动提醒；超 95% 自动触发压缩，群里通知"已自动压缩 @产品A_bot 的上下文"
多项目管理可观测性
　
文件发送
bot 完成任务后能把生成的文件（图片、PDF、代码文件）直接发到群里，团队不用去服务器看
多人协作
　
Reaction 快捷操作
对 bot 结果 react 🔄 重跑、📌 置顶到今日重要结果、❌ 取消正在跑的任务。不用打字就能操作
效率多人协作

审批增强

　
审批委托
/delegate @Leo 2h — 接下来 2 小时 Leo 可以代批 approve 请求，到期自动回收。admin 不在时不卡流程
多人协作高优先
　
多人审批（多签）
敏感项目可配置需要 2 人同时批准。按钮上显示"1/2 已批准"，防止单人误操作
多人协作安全

v0.5 — 深度协作 PLANNED

目标：围绕团队协作深挖，让群聊工作台成为不可替代的工作方式

团队感知

　
智能摘要 /digest
离开几小时回来，一条命令生成摘要：完成了什么、失败了什么、谁在等你反馈、哪些结果没人确认
多人协作异步协作
　
谁在做什么 /who
实时显示团队工作状态：Leo 正在用 @产品A_bot（12分钟前）、Momo 在 @产品C_bot（刚刚）、其他人空闲
多人协作
　
通知订阅 /watch
每人可选择只关注特定 bot 的结果通知，其他 bot 静音。解决消息量大时的信噪比问题
多人协作
　
关键结果 @提及
bot 结果包含 error/failed/需要确认等关键词时，自动 @相关成员，重要信息不被淹没
多人协作

协作流程

　
任务接力 /handoff
Leo 让 @产品A_bot 改完 API 后，/handoff @Momo "样式需要调整" — Momo 收到通知，带完整上下文继续
多人协作独占功能
　
跨项目联动
"@产品A_bot 改完后通知 @产品B_bot 更新前端对接" — bot 间任务链，结果对全员可见
多项目管理独占功能
　
共享命令模板
每个 bot 预配常用操作按钮：[跑测试] [代码审查] [更新文档]。新人不用记命令，点按钮就行
多人协作效率
　
快捷指令面板 /quick
弹出按钮面板：[全部跑测试] [全部检查文档] [刷新仪表盘]。一键批量操作多个项目

不做的事

方向原因
Web 全功能 dashboard投入大、维护重。按钮菜单 + 仪表盘 + 状态页已够用
云托管版设计为本地运行。云版涉及安全、计费、基础设施，不是我们的方向
代替终端的完整交互体验我们是调度层，不是终端模拟器。复杂调试还是应该去终端

决策原则

每个新功能都要回答：

  • 这个功能让 多人在同一个群里协作 变得更好了吗？
  • 这个功能让 多项目集群管理 变得更顺畅了吗？
  • 如果两个都不是，不做。
